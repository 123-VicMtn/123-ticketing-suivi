# Comment contribuer à 123-ticketing

Merci de prendre le temps d'aider ! Ce guide explique comment participer efficacement.

## Signaler un bug

Avant de créer une discussion, vérifiez qu'il n'existe pas déjà dans [🐛 Bugs reportés](https://github.com/123-VicMtn/123-ticketing-suivi/discussions/categories/bugs-report%C3%A9s).

**Pour chaque bug, précisez :**

```
**Description**
[Décrivez le problème en une phrase]

**Étapes pour reproduire**
1. Aller sur...
2. Cliquer sur...
3. Observer...

**Comportement attendu**
[Ce qui devrait se passer]

**Comportement observé**
[Ce qui se passe réellement]

**Environnement**
- Appareil : [iPhone 15 / MacBook / PC Windows]
- Navigateur : [Chrome 124 / Safari / Firefox]
- Version de l'app : [v0.1 ou "dernière version"]

**Capture d'écran** (si possible)
```

## Proposer une idée

Ouvre une discussion dans [💡 Idées & suggestions](https://github.com/123-VicMtn/123-ticketing-suivi/discussions/categories/id%C3%A9es-suggestions) avec :
- Le problème que ça résoudrait
- Votre proposition concrète
- Pourquoi c'est utile (pour vous ou d'autres)

## Participer aux sessions de test

Chaque release a un fil dans [🧪 Sessions de test](https://github.com/123-VicMtn/123-ticketing-suivi/discussions/categories/sessions-de-test). Lisez le post, testez ce qui est demandé, et répondez dans le fil ou créez des discussions séparées pour les bugs.

## Ce qui arrive à tes retours

1. Je lis tout sous 48h
2. Les bugs confirmés deviennent des Issues GitHub et vous serez informés sur le suivi.
3. Les idées retenues sont marquées comme "Answer" dans la discussion
4. Le récap hebdo liste ce qui a été traité

## Questions

Ouvre une discussion dans [💬 Général](https://github.com/123-VicMtn/123-ticketing-suivi/discussions/categories/g%C3%A9n%C3%A9ral) — pas d'email SVP, ça permet à tout le monde de profiter de la réponse.
